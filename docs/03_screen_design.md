## 3. 画面ごとのワイヤー概要

### SC-00 ログイン
```
┌─────────────────────────┐
│      LogiCore WMS       │
│   WAREHOUSE MANAGEMENT  │
│                         │
│   社員番号              │
│  [ E00123        ]      │
│   パスワード（数字5桁）│
│  [ ● ● ● ● ●     ]      │
│                         │
│   [    ログイン    ]    │
│                         │
│   認証エラー時は赤枠    │
└─────────────────────────┘
```
- **社員番号** 入力欄: IME オフ、英数のみ許可、最大20桁
- **パスワード** 入力欄: `type="password"`、`inputmode="numeric"`、
  `pattern="[0-9]{5}"`、最大5桁。モバイルではテンキーを自動表示
- 失敗時のメッセージ:
  - 入力誤り: `社員番号またはパスワードが違います`
  - 一時ロック中: `ログインが一時的にロックされています。{残り時間}後に再試行してください`
  - 強制ロック: `アカウントがロックされています。管理者に連絡してください`
- 試行回数は画面に表示しない（攻撃者への情報を与えない）

### SC-01 ダッシュボード
- 4 枚の KPI カード（総在庫 / 本日入庫 / 本日出庫 / アラート）
- 7日間トレンドチャート（入庫=緑 / 出庫=青）
- アラート一覧（残少・期限接近・受付待ち）
- 最近の操作履歴テーブル

### SC-02 商品マスタ一覧
- 上部: 検索ボックス / 新規ボタン（admin のみ）/ CSV ボタン
- テーブル列: SKU / 商品名 / JAN / 単位 / 入数 / 保管条件 / 最低在庫 / 操作
- 行クリックで編集モーダルへ（または詳細画面）

### SC-06 入庫一覧
- タブ: `全て | 受付待ち | 入荷済 | 検品済 | 棚入済`
- テーブル列: 伝票番号 / 仕入先 / 予定日 / 明細数 / 予定総数 / ステータス / 操作
- 新規ボタン、CSV インポート

### SC-07 入庫詳細・検品
- ヘッダ: 伝票番号 / 仕入先 / 予定日 / ステータス
- 明細テーブル: 行ごとに `予定数 / 実数入力（input）/ ロケーション（scanで入力）`
- 下部ボタン: `検品完了` → `棚入完了` → `入庫完了`

### SC-08 / SC-09 出庫
入庫とほぼ同等。出庫ではピッキング画面に **FIFO 推奨ロット** を表示する。

### SC-10 在庫一覧
- フィルタ: SKU / 商品名 / ゾーン / 保管条件 / 残少のみ / 期限 30日以内
- テーブル列: SKU / 商品名 / ロケ / ロット / 期限 / 在庫 / 引当中 / 利用可能
- 低在庫行は左端に赤バー、期限接近は黄バー
- CSV エクスポートボタン

### SC-11 ロケーション移動（新規）
```
┌───── ロケーション移動 ───────────────┐
│ 移動元ロケ: [A-01-03-2-B ▢ scan]     │
│ 商品:       [SKU / 商品名 表示]       │
│ ロット:     [LOT-20260401]            │
│ 移動数量:   [   50 ] / 在庫 200       │
│ 移動先ロケ: [B-02-01-1-A ▢ scan]     │
│                                      │
│            [キャンセル] [移動実行]   │
└──────────────────────────────────────┘
```
- スキャンフォーカスは「移動元 → SKU 確認 → 移動先」の順に自動遷移
- 実行時に `inventory` を更新し、トースト `移動完了 +50 B-02-01-1-A`

### SC-12 棚卸一覧
- テーブル列: 棚卸名 / 対象ゾーン / 予定日 / ステータス / 進捗率 / 操作
- 新規棚卸: 対象ゾーン・期日・担当者 を指定。作成時に対象在庫のスナップショットを `stocktake_items` に生成

### SC-13 棚卸カウント（モバイルファースト）
```
┌──── 棚卸: 2026年4月A棚 ─────┐
│ ▶ 進捗 12 / 48              │
│                             │
│ ロケ: A-01-03-2-B           │
│ SKU: P-00123                │
│ 商品: ブランドX 500ml       │
│ ロット: LOT-20260301        │
│ 期限: 2027-03-01            │
│ システム在庫:  200          │
│ 実数:   [   198  ] 入力     │
│                             │
│ [ スキャンでSKU確認 ]       │
│ [ 前の行  ] [ 次の行 ▶ ]    │
└─────────────────────────────┘
```
- 差異が出た場合はフィールドを黄色くハイライト
- 全行カウント済みになると `[ レビューへ ]` ボタンが有効化

### SC-14 棚卸差異・確定
- テーブル: ロケ / SKU / システム / 実数 / 差異 / 備考
- 差異のある行のみ表示するトグル
- `[棚卸確定]`（admin のみ）→ 在庫調整トランザクション実行

### SC-15 スキャン共通モーダル
- 画面下 2/3 をカメラビュー、上 1/3 に認識結果と対象コンテキスト（入庫/移動/棚卸）を表示
- 物理ハンディは不可視 input にフォーカスを当てる実装
- バーコード規格: EAN-13 / Code-128 / QR Code

### SC-16 レポート/CSV
- 出力候補:
  - 在庫一覧（現在）
  - 入庫履歴（期間指定）
  - 出庫履歴（期間指定）
  - 棚卸差異（棚卸選択）
- UTF-8 BOM 付き CSV、ファイル名 `wms_<report>_<yyyymmdd>.csv`

### SC-17 ユーザー管理
- 一覧: 社員番号 / 表示名 / ロール / 最終ログイン / ロック状態
- admin のみアクセス可
- 新規ユーザー発行時に **社員番号 + 初期パスワード（数字5桁）** を設定
- ロール変更 / パスワードリセット（初期値に戻す）/ ロック解除 / 無効化

## 4. コンポーネント共通ルール

| 要素 | 規則 |
|------|------|
| ボタン一次 | `.btn.btn-p` アクセント色背景 |
| ボタン二次 | `.btn.btn-g` 透明 + ボーダー |
| ボタン破壊 | `.btn.btn-d` 赤 |
| バッジ | 状態を `bg/br/by/bb/bgr` で色分け |
| トースト | 画面右下、3 秒で自動消去 |
| 空状態 | アイコン + 説明文 + 主アクション |
| ローディング | スケルトン行 or 中央スピナー |

## 5. 色・状態ガイド

| ステータス | 色 | クラス |
|------------|----|--------|
| 完了 / 正常 | `#00e5a0` (green) | `bg` |
| 進行中 / 注意 | `#ffd32a` (yellow) | `by` |
| エラー / 残少 | `#ff4757` (red) | `br` |
| 情報 / 入庫 | `#4d9fff` (blue) | `bb` |
| 中立 | `#7a8099` (gray) | `bgr` |

## 6. スマホ最適化メモ

- タップ領域は最低 44×44px
- 棚卸カウント・スキャン画面は片手操作前提で下部にアクション集約
- 下タブは safe-area-inset-bottom を考慮
- 入力時にソフトキーボードが出ても主要 UI がスクロール可能になっていること
