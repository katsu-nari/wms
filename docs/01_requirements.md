## 1. 背景と目的

倉庫業務で発生する「入庫 / 出庫 / 在庫管理 / ロケーション管理 / 棚卸 / ロケーション移動」を
一気通貫でデジタル化し、紙・Excel運用からの脱却とリアルタイム在庫可視化を実現する。

既に `wms_supabase.html` として単一HTML＋Supabaseで稼働するプロトタイプが存在する。
本要件定義は、そのプロトタイプをベースに **本番運用可能なWMS** へ育てるための指針を定める。

## 2. スコープ

### 2.1 実装する機能（In Scope）

| # | 機能カテゴリ | 概要 |
|---|--------------|------|
| F-01 | 認証・権限管理 | メール+パスワードログイン、ロールベース権限 |
| F-02 | ダッシュボード | KPI・アラート・最近の操作履歴・入出庫トレンド |
| F-03 | 商品マスタ管理 | SKU・商品コード・JANコード・規格・保管条件 |
| F-04 | ロケーション管理 | 棚・通路・ゾーンの階層管理、ロケーションコード発番 |
| F-05 | 入庫処理 | 入庫予定登録、入荷検品、棚入れ、入庫完了 |
| F-06 | 出庫処理 | 出庫指示、ピッキング、出荷検品、出庫完了 |
| F-07 | 在庫一覧 | ロット・期限・ロケーション別の在庫照会 |
| F-08 | ロケーション移動 | 在庫を別ロケーションへ移動（補充・集約・整理） |
| F-09 | 棚卸 | 棚卸指示、実数カウント、差異登録、棚卸確定 |
| F-10 | バーコード/QRスキャン | カメラ/ハンディスキャナ入力による検品・移動・棚卸 |
| F-11 | 帳票・CSVエクスポート | 在庫・入出庫・棚卸差異のCSV出力 |

### 2.2 スコープ外（Out of Scope）

- クライアント（荷主）管理機能 — 現プロトタイプからは**削除**する
- 配送・TMS 連携（運送会社との API 連携）
- 会計・原価計算（WMS 範囲外）
- 多言語対応（当面は日本語のみ）
- WMS-WCS 連携（自動倉庫制御）

## 3. ユーザーとロール

| ロール | 想定ユーザー | 主な権限 |
|--------|--------------|----------|
| `admin` | 倉庫管理者 | 全機能。マスタ編集、棚卸確定、ユーザー管理 |
| `operator` | 現場オペレータ | 入出庫、スキャン、ロケーション移動、棚卸カウント |
| `viewer` | 監査・閲覧者 | 在庫・履歴の閲覧のみ。編集不可 |

権限マトリクスの詳細は `04_architecture.md` 参照。

## 4. 機能要件

### F-01 認証・権限管理
- Supabase Auth のメール+パスワード認証を利用する
- ログイン成功後、`profiles` テーブルから `role` を取得しクライアントに保持
- RLS（Row Level Security）で DB レベルの権限制御を行う
- 未ログイン時は `/login` にリダイレクト

### F-02 ダッシュボード
- KPI: 総在庫数、本日入庫数、本日出庫数、アラート件数
- アラート: 在庫残少、期限切れ接近、入庫受付待ち
- 過去7日間の入出庫トレンド（棒グラフ）
- 最近の操作履歴（入庫・出庫 混在）

### F-03 商品マスタ管理
- 必須項目: 商品コード（SKU）、商品名、単位、入数
- 任意項目: JANコード、規格、保管条件（常温/冷蔵/冷凍/危険物）、最低在庫数、有効期限管理要否
- 一覧検索（商品コード・商品名・JAN部分一致）
- 新規登録 / 編集 / 論理削除

### F-04 ロケーション管理
- 階層: ゾーン > 通路 > 棚 > 段 > ビン
- ロケーションコード例: `A-01-03-2-B`
- 属性: 保管条件、容量、ピッキング優先度、有効/無効
- 一覧 + 新規登録 + 編集

### F-05 入庫処理
- 入庫予定登録（手入力 / CSV インポート）
- ステータス遷移: `受付待ち → 入荷済 → 検品済 → 棚入済(完了)`
- 検品時: 予定数量と実数量の差異を記録
- 棚入れ時: ロケーションをスキャンして確定
- 完了時に `inventory` テーブルへ在庫加算 + `inventory_movements` に履歴追加

### F-06 出庫処理
- 出庫指示登録（手入力 / CSV インポート）
- ステータス遷移: `指示待ち → ピッキング中 → 検品済 → 出荷済(完了)`
- ピッキング時: ロケーション＋SKUをスキャンして引当
- FIFO（先入れ先出し）を入庫日/期限ベースで推奨
- 完了時に `inventory` 減算 + `inventory_movements` 記録

### F-07 在庫一覧
- 検索軸: SKU、商品名、ロケーション、ロット、期限、保管条件
- 行: SKU × ロット × ロケーション で一意
- 期限切れ / 残少 の行を赤/黄ハイライト
- CSV エクスポート

### F-08 ロケーション移動（新規）
- 対象在庫を選択し、移動元 → 移動先ロケーションを指定
- 数量指定（部分移動可）
- スキャンで元ロケ→SKU→移動先ロケ を順次確認
- 完了時に `inventory` の `location_id` を更新 + `inventory_movements` に `type='move'` 記録

### F-09 棚卸（新規）
- 棚卸指示登録: 対象ロケーション範囲・期日・担当者
- カウント画面: 指示に紐づくロケーションを順に開き、実数を入力
- 差異レポート: システム在庫 vs 実数、差異行のみ抽出
- 管理者が**確定**すると、差異分を `inventory` に反映し `inventory_movements` に `type='adjust'` 記録
- 棚卸中のロケーションは他の入出庫からロックされる

### F-10 バーコード/QRスキャン
- Web（`BarcodeDetector` API または `ZXing-js`）で SKU/ロケーション/伝票番号を読み取り
- スキャン結果はコンテキスト（入庫/出庫/移動/棚卸）に応じて処理
- 物理ハンディスキャナ（キーボードエミュレート型）にも対応

### F-11 帳票・CSVエクスポート
- 在庫一覧 CSV
- 入庫履歴 CSV（期間指定）
- 出庫履歴 CSV（期間指定）
- 棚卸差異 CSV（棚卸ID指定）
- 文字コード: UTF-8 (BOM付き、Excel 互換)

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | 一覧画面 1000 行を 2 秒以内に描画 |
| 同時接続 | 50 名同時利用を想定 |
| 可用性 | Supabase の SLA に準拠（99.9%） |
| セキュリティ | RLS 必須、HTTPS 必須、API Key は匿名キーのみフロント配置 |
| 監査 | 全在庫変動を `inventory_movements` に不変記録 |
| 対応ブラウザ | Chrome / Edge / Safari 最新 2 バージョン |
| モバイル | スマートフォン縦画面 375px 以上をサポート |
| データ保持 | 取引履歴 3 年以上、マスタは論理削除のみ |

## 6. 用語集

| 用語 | 意味 |
|------|------|
| SKU | Stock Keeping Unit。在庫管理単位 |
| ロケーション | 保管場所（棚のビン単位） |
| 棚卸 | 実在庫を数えてシステム在庫と突合する作業 |
| 引当 | 出庫指示に対して特定の在庫を割り当てる処理 |
| FIFO | First In First Out。先入れ先出し |
| RLS | Row Level Security。Postgres の行レベル権限制御 |
